## Установка

### Вариант 1 — Веб-патчер (рекомендуется)

> Ничего ставить не нужно — всё работает в браузере.

1. Открой **[веб-патчер](https://itzcaek.github.io/F7CK-funpay-lite-bot/)**.
2. Возьми оригинальный CRX — [из Chrome Web Store](https://chromewebstore.google.com/detail/funpay-lite-bot/amicfiagmpbgfiiopieeemlkblfeeeip) или [из архива](archive/v2.0.6/funpay-lite-bot-v2.0.6-original.crx).
3. Перетащи `.crx` в окно патчера и дождись сборки.
4. Скачай готовый `funpay-lite-bot-patched.zip` и распакуй его.
5. Открой `chrome://extensions/` и включи **Режим разработчика**.
6. Нажми **«Загрузить распакованное расширение»** и выбери распакованную папку.
7. Отключи оригинальное расширение и перезагрузи **FunPay**.

### Вариант 2 — Python

```bash
git clone https://github.com/itzcaek/F7CK-funpay-lite-bot.git
cd F7CK-funpay-lite-bot
python patcher.py
```

Скрипт применит те же патчи к CRX и сохранит распакованную папку рядом.

## FAQ

<details>
<summary><b>Это безопасно? Куда уходят мои данные?</b></summary>
<br>
Никуда. Патчинг происходит целиком в браузере (или локально через Python). CRX не загружается ни на какой сервер.
</details>

<details>
<summary><b>Почему расширение нужно ставить в режиме разработчика?</b></summary>
<br>
Патченная сборка не подписана Chrome Web Store, поэтому устанавливается как распакованное расширение через <code>chrome://extensions/</code>.
</details>

<details>
<summary><b>Обновится ли патч под новую версию расширения?</b></summary>
<br>
Если структура кода не изменилась — да. При крупном рефакторинге со стороны разработчиков патчи нужно будет обновить.
</details>

<details>
<summary><b>В чём разница между веб-версией и Python?</b></summary>
<br>
Результат одинаковый. Веб-патчер удобнее (ничего не ставить), Python — для тех, кто хочет запускать из терминала или автоматизировать.
</details>
